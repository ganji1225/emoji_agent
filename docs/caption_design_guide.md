# キャプション設計ガイド

VoiceDesign モデル向けの「声の説明文」を効果的に書くための実践ガイド。
これを意識すると音声品質が劇的に上がる。

---

## 0. キャプションとは

VoiceDesign モデルは、参照音声の代わりに**テキストで声を指定**できるモデル。

```
infer.py
  --no-ref                   ← 参照音声を使わない
  --caption "若い女性が..."  ← 代わりにキャプションで声を指定
```

このキャプション文の品質が、出力音声の品質を直接決める。

---

## 1. 良いキャプションの基本構造

```
{年齢表現}が、{行動描写}しながら、{声質指示}で話している。
```

### 各要素の役割

| 要素 | 効果 |
|------|------|
| 年齢表現 | 声の高さ・成熟度を決める |
| 行動描写 | テンポ・リズム・抑揚を変える |
| 声質指示 | 音の柔らかさ・響きを調整 |

---

## 2. 年齢表現で声質をコントロール

| 表現 | 出る声の傾向 |
|------|------------|
| 幼い女の子 | 高く、舌足らず、たどたどしい |
| 少女 | 中間。明るく若々しい |
| 若い女性 | 大人っぽく落ち着き |
| 大人の女性 | 低めで成熟、艶がある |
| 落ち着いた女性 | より低く、余裕がある |

### 例
```
✅ 「幼い女の子が、無邪気に笑いながら話している。」  → 高い舌足らず声
✅ 「若い女性が、リラックスして話している。」      → 標準的な声
✅ 「大人の女性が、艶やかにささやいている。」      → 落ち着いた色気
```

---

## 3. 行動描写で抑揚・テンポを変える

### 効果一覧

| 行動描写 | 出る声の変化 |
|---------|------------|
| 一生懸命 | 声にハリが出る、芯が強くなる |
| たどたどしい | 区切りが多く、たどたどしく話す |
| 慌てる / 焦って | テンポが上がる、息が荒くなる |
| 穏やかに | 柔らかく、ゆったりしたテンポ |
| 恥ずかしそうに | 声が小さくなる、もじもじ感 |
| 涙を堪えて | 声が震える、詰まる |
| 自信を持って | 強く、はっきりした発音 |
| 甘えるように | 語尾が伸びる、媚びた響き |

### NG vs OK 比較

❌ **NG**：
```
「高い声で話している」
```
→ 抽象的すぎて変化が出にくい。声の高さだけでキャラが薄い。

✅ **OK**：
```
「一生懸命だけど少しドジっ子っぽく話している」
```
→ 具体的な行動・性格が入って、声に**個性が出る**。

❌ **NG**：
```
「悲しい声で話している」
```

✅ **OK**：
```
「涙を堪えきれず声を詰まらせながら、感情を込めて話している」
```
→ 「涙を詰まらせる」という行動が入ることで、震えや息継ぎが自然に出る。

## 4. CFG値ガイド

CFG = キャプションをどれだけ強く反映するかの強度。

| CFG | 用途 | 特徴 |
|-----|------|------|
| 3.0 | 日常・自然体 | キャプションの影響弱め、自由度高い |
| 5.0 | 中程度の感情 | 恥じらい・甘え・動揺に最適 |
| 6.0 | 極限 intensity 専用 | **絵文字4+4と組み合わせる**。感情表現↑＋かすれ抑制 |
| 7.0 | 強い感情 | 喘ぎ・叫び・崩壊。ノイズとのバランス良 |
| 8.0 | **非推奨** | かすれ・ノイズ多発。代わりに絵文字4+4×CFG6.0を使う |

### 絵文字数 × CFG の組み合わせ

| 絵文字数 | 推奨CFG | 用途 |
|---------|--------|------|
| 2+1 | 3.0 | 弱（日常・自信） |
| 2-3+1 | 5.0 | 中（恥じらい・困惑） |
| 3+2 | 7.0 | 強（喘ぎ・絶頂） |
| 4+4 | 6.0 | 極限（崩壊・連続絶頂） |

詳細は [`emoji_pattern_reference.md`](./emoji_pattern_reference.md) を参照。

## 5. guidance_mode の使い分け（2026-04-25 確定）

`--cfg-guidance-mode` は3種類。聴き比べテストで以下の結論：

| モード | 評価 | 推奨用途 |
|--------|------|---------|
| **independent** | 平坦な感情 ❌ | 廃止・互換性のみ保持 |
| **joint** | 声質変化が滑らか ✅ | **静かなシーン**（日常・囁き・余韻・甘い系） |
| **alternating** | 抑揚・リズム良好 ✅ | **感情強いシーン**（喘ぎ・絶頂・崩壊・泣き） |

### プロファイル別マッピング（default_captions.json）

| プロファイル | mode |
|-------------|------|
| 日常会話 | joint |
| 甘い囁き | joint |
| 余韻・安堵 | joint |
| 恥じらい | alternating |
| 積極的・甘え | alternating |
| 興奮・高まり | alternating |
| 絶頂・叫び | alternating |
| 極限・崩壊 | alternating |
| 泣き・感動 | alternating |

---

## 6. num_steps（拡散ステップ数）

| 値 | 用途 |
|----|------|
| 30 | 高速プレビュー（旧デフォ） |
| **40** | **本番推奨**（2026-04-25 から） |
| 50+ | 高品質確認用、生成時間が長くなる |

40 にすると 30 より明らかに細部の表現力が上がる。生成時間は約30%増。

## 7. プロファイル別キャプション例

`D:/irodori/agents/default_captions.json` で管理されている公式プロファイル：

### 日常会話（CFG=3.0, joint, 40steps）
```
若い女性が、リラックスした自然な口調で、のんびり明るく話している。
```

### 甘い囁き（CFG=5.0, joint, 40steps）
```
若い女性が、耳元で甘く囁くように、とろけた柔らかい声で息を漏らしながら話している。
```

### 恥じらい（CFG=5.0, alternating, 40steps）
```
若い女性が、恥ずかしそうにもじもじしながら、言葉を選ぶように小さな声で話している。
```

### 積極的・甘え（CFG=5.0, alternating, 40steps）
```
若い女性が、自信を持って甘く誘うように、距離を詰めながらからかうように話している。
```

### 興奮・高まり（CFG=7.0, alternating, 40steps）
```
若い女性が、息を荒くしながら、感情が抑えきれず声が震えて話している。
```

### 絶頂・叫び（CFG=6.0, alternating, 40steps）
```
若い女性が、感情が限界に達して、涙声で息を詰まらせながら叫んでいる。
```

### 極限・崩壊（CFG=6.0, alternating, 40steps）
```
若い女性が、完全に理性を失い、声が裏返りながら荒い息で絶叫している。
```

### 余韻・安堵（CFG=3.0, joint, 40steps）
```
若い女性が、ほっとしたように力が抜けて、穏やかに息をつきながらゆっくり話している。
```

### 泣き・感動（CFG=5.0, alternating, 40steps）
```
若い女性が、涙を堪えきれず声を詰まらせながら、感情を込めて話している。
```

## 9. やってはいけないこと

### ❌ NG パターン

1. **抽象的すぎる**
   ```
   「いい感じの声」「美しく」「綺麗に」
   ```
   → モデルが解釈できない、効果が薄い

2. **矛盾する指示**
   ```
   「明るく元気で穏やかに静かに話している」
   ```
   → どっち方向にも振れず、平坦になる

3. **キャプション中に絵文字を入れる**
   ```
   「🥵 喘ぎながら話している 💦」
   ```
   → キャプションは text 用、絵文字は text_with_emoji 用に分ける

4. **長すぎ**（200文字超）
   ```
   「色々な特徴を持った若い女性で、髪が長くて...」
   ```
   → 後半が無視される、シンプルが正義

5. **男性指示**
   ```
   「若い男性が...」
   ```
   → モデルは女性声で学習されているため、男性声は出ない

## 10. デバッグ Tips

### 思った声と違う場合

1. **CFG を1段階上げる**（3.0→5.0、5.0→7.0）→ キャプションがより強く反映される
2. **行動描写を具体化**（「優しく」→「優しく語りかけるように、安心させながら」）
3. **年齢表現を見直す**（「女性」→「若い女性」「大人の女性」など）
4. **guidance_mode を切り替える**（joint ↔ alternating）

### 候補ガチャで安定しない場合

1. **num_candidates=3** で3つ生成して聴き比べ（標準）
2. **同じ seed** を指定して再現性確保
3. **絵文字数を減らす**（4個→3個で安定性UP、表現力DOWN）
